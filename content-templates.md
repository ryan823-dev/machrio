# Machrio内容创作模板与规范

## 📝 内容类型规范

### 1. Knowledge Center文章模板

**适用场景：**
- What is X? 类定义性内容
- 类别介绍和分类说明
- 行业概念解析

**结构要求：**
```
H1: 核心问题标题（包含目标关键词）
Quick Answer段落（2-3句话直接回答）
详细解释部分（H2层级展开）
应用场景举例
FAQ常见问题（3-5个）
CTA引导到相关产品或类目
```

**写作要点：**
- 开头直接回答核心问题
- 使用简单易懂的工业术语
- 每个H2段落控制在200-400字
- FAQ要针对真实搜索意图
- 结尾引导到商业转化

### 2. 商业落地页模板

**适用场景：**
- Buy X Online
- Shop X Products
- Industrial X Supplies

**结构要求：**
```
H1: 商业导向标题
价值主张段落
产品类别展示
服务优势说明
采购流程介绍
CTA按钮（询盘/浏览产品）
```

**写作要点：**
- 突出平台优势（DDP、MOQ、一站式）
- 明确服务承诺
- 展示品类丰富度
- 强化信任背书

### 3. FAQ知识页模板

**适用场景：**
- 对比类问题
- 定义澄清类
- 选择指导类

**结构要求：**
```
H1: 问题式标题
简短答案（1-2段）
详细解释（如有必要）
应用场景
总结要点
```

### 4. Glossary词条模板

**适用场景：**
- 专业术语定义
- 缩写词解释
- 技术概念说明

**结构要求：**
```
H1: What is X?
定义卡片（突出显示）
详细解释内容
相关标准/认证（如适用）
应用场合
相关术语链接
```

## 🔍 关键词优化规范

### 标题优化
- 主关键词出现在H1开头
- 长尾词自然融入
- 长度控制在50-60字符

### 内容优化
- 主关键词密度：1-2%
- 相关词自然分布
- 每段落包含相关语义词
- 使用同义词和变体

### 链接策略
- 内部链接：指向相关产品类目
- 锚文本多样化
- 每篇文章包含2-4个内链
- 外链指向权威来源（适度）

## 🎯 各主题内容规划

### MRO主题簇

**1. What Is MRO in Manufacturing?**
- 目标词：what is mro, mro meaning, mro in manufacturing
- 结构：定义 → 分类 → 应用场景 → FAQ
- 字数：800-1200字

**2. What Are MRO Products?**  
- 目标词：what are mro products, mro products, industrial mro
- 结构：产品分类详解 → 各类举例 → 采购建议
- 字数：1000-1500字

**3. Buy Industrial MRO Supplies Online**
- 目标词：online mro supplies, buy industrial mro supplies
- 结构：平台优势 → 产品类别 → 服务承诺 → CTA
- 字数：600-800字

**4. MRO vs OEM vs Production Materials**
- 目标词：mro meaning manufacturing, material mro
- 结构：三者定义对比 → 应用区别 → 选择建议
- 字数：700-1000字

### Respirator主题簇

**1. Types of Respirators Explained**
- 目标词：types of respirators, respirator types
- 结构：分类介绍 → 各类特点 → 选择指南

**2. PAPR vs N95**
- 目标词：papr vs n95
- 结构：对比表格 → 优缺点分析 → 应用场景

**3. Respirator APF Guide**
- 目标词：respirator apf, apf respirator
- 结构：APF解释 → 数值对照表 → 选择建议

## 📊 内容质量检查清单

□ 标题包含主关键词
□ 开头段落直接回答核心问题
□ 使用适当的H2-H3结构
□ 包含相关内部链接（2-4个）
□ FAQ部分针对性强
□ CTA引导自然合理
□ 字数符合要求
□ 语言专业但易懂
□ 无语法拼写错误

## 🔄 创作流程

1. **关键词研究** - 确认目标词搜索量和竞争度
2. **大纲制定** - 按模板结构规划内容框架
3. **初稿撰写** - 按大纲填充内容
4. **优化调整** - 检查关键词密度和可读性
5. **内部审核** - 确保技术准确性
6. **CMS发布** - 设置正确的元数据和分类
7. **效果跟踪** - 监控排名和流量表现
