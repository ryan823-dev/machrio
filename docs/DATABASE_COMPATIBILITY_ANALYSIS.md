# 数据库格式不一致对业务影响分析

## 核心问题

**担心：MongoDB 和 PostgreSQL 格式不一致，导致产品上传出问题**

## 实际情况分析

### ✅ 好消息：Payload CMS 已经处理了格式转换

**关键发现：**

查看 `src/payload/payload.config.ts` 配置：

```typescript
// 数据库配置
db: process.env.USE_POSTGRES === '1' && process.env.DATABASE_URI
  ? postgresAdapter({ ... })  // PostgreSQL
  : mongooseAdapter({ ... })  // MongoDB
```

**这意味着：**
- Payload CMS 作为 ORM 层，自动处理数据库格式转换
- 你上传产品时，代码是一样的
- CMS 会自动将数据转换为对应数据库的格式

## 潜在风险点分析

### ⚠️ 风险 1：直接数据库操作

**如果你直接写 SQL 或 MongoDB 查询：**

```javascript
// ❌ 错误做法：直接使用数据库特定语法

// MongoDB 查询
db.products.find({ sku: 'SEN-2024-001' })

// PostgreSQL 查询
SELECT * FROM products WHERE sku = 'SEN-2024-001'
```

**问题：**
- 这种代码在另一种数据库会报错
- 混合模式下会出问题

**实际影响：**
- ⚠️ 如果你修改了数据库相关代码
- ⚠️ 可能在不同环境表现不同

**解决方案：**
- ✅ 使用 Payload CMS 的 API
- ✅ 不要直接操作数据库

---

### ⚠️ 风险 2：数据迁移脚本

**如果你手动迁移数据：**

```javascript
// MongoDB 格式
{
  _id: ObjectId("..."),
  categories: [ObjectId("...")]
}

// PostgreSQL 格式
{
  id: "uuid-string",
  categories: ["uuid-string"]
}
```

**问题：**
- ID 格式不同（ObjectId vs UUID）
- 关系字段格式不同

**实际影响：**
- ⚠️ 迁移脚本需要特殊处理
- ⚠️ 但我们已经解决了（有成熟的迁移脚本）

**解决方案：**
- ✅ 使用现有的迁移脚本
- ✅ 不要手动写迁移代码

---
